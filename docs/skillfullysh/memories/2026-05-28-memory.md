# 2026-05-28 Memory

- GitHub import candidate rows now display as `<repo>/<skill name>` instead of `<repo> - <skill name>`.
- Candidate-level import failures, including schema validation failures such as `Validation failed for steps: Attributes are missing in your schema`, now stay in the import modal as red disabled rows with the failure reason attached to the failed skill.
- The import POST response now returns updated candidates and persists failed candidates into the cached import session, while account/session-level failures remain modal-level errors.
- GitHub import discovery no longer aborts the GET request when existing-import checks, repository upserts, or import-session cache writes fail; those errors become warnings, and validation messages that name a discovered skill are attached to that skill row.
- Validation warnings already attached to a skill row are removed from the global warning list, and selectable import rows/checkboxes now show the pointer cursor while disabled rows show not-allowed.
- Diagnosed the imported `marketingskills/skills/video` truncation as a dashboard markdown editor issue, not an import issue: the stored `SKILL.md`, `references/ai-video-prompting.md`, and `evals/evals.json` were complete, but MDXEditor lacked fenced code block support. Enabled code block and CodeMirror plugins and added coverage that frontmatter stripping preserves body horizontal rules and fenced code blocks.
